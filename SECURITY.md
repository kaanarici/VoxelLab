# Security Policy

## Supported Version

Security fixes target the current `main` branch and the latest tagged release.
Older releases are not maintained.

## Report a Vulnerability

Do not open a public issue for an undisclosed vulnerability.

Use the **Email** contact on the
[repository owner's website](https://kaanarici.com/) to send a private report.
Do not include patient data, credentials, private service URLs, or third-party
data you are not authorized to share.

Include:

- the affected component and version or commit
- a minimal reproduction or clear description
- the expected security impact
- any suggested mitigation, if available

There is no guaranteed response time. Please allow the maintainer to assess and
fix a report before public disclosure.

## Scope

This policy covers the VoxelLab repository, packaged desktop application, local
server, and documented processing workflows. It does not authorize testing
third-party services, public deployments, accounts, systems, or data you do not
own or have permission to test.

VoxelLab is not for clinical use. This policy does not replace an institution's
rules for patient data, regulated systems, or medical safety.

## Safe Harbor

Good-faith research that respects the scope above, avoids privacy violations,
and gives the maintainer a reasonable opportunity to address a report is
welcome.
